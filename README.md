# Варіант індивідуального завдання

Цей розділ містить розрахунок варіанта та детальний опис функціональних вимог до веб-застосунку та інфраструктури.

## Розрахунок варіанта
Розрахунок проведено на основі номера залікової книжки (або порядкового номера) **N = 6**.

| Параметр | Формула | Розрахунок | Значення |
| :--- | :--- | :--- | :--- |
| **V2** (Конфігурація та БД) | `(N % 2) + 1` | `(6 % 2) + 1 = 1` | **1** |
| **V3** (Тематика застосунку) | `(N % 3) + 1` | `(6 % 3) + 1 = 1` | **1** |
| **V5** (Порт застосунку) | `(N % 5) + 1` | `(6 % 5) + 1 = 2` | **2** |

## Опис індивідуального завдання

### 1. Веб-застосунок: Notes Service (`mywebapp`)
Простий сервіс для керування текстовими нотатками.

**Структура об'єкта нотатки:**
* `id` (унікальний ідентифікатор)
* `title` (заголовок)
* `content` (вміст нотатки)
* `created_at` (дата та час створення)

**API ендпоінти:**
* `GET /notes` — повертає список усіх нотаток (тільки `id` та `title`).
* `POST /notes` — створює нову нотатку (приймає `title`, `content`).
* `GET /notes/<id>` — повертає повний вміст конкретної нотатки.

### 2. Технічні характеристики
* **База даних:** MariaDB (доступ обмежений лише локальним інтерфейсом `127.0.0.1`).
* **Порт застосунку:** `5200`.
* **Спосіб конфігурації:** Аргументи командного рядка (CLI arguments) для передачі порту та параметрів підключення до БД.
* **Формати відповідей:** Підтримка заголовка `Accept`:
    * `application/json` — дані у форматі JSON.

### 3. Системні вимоги та розгортання
* **Reverse Proxy:** Nginx (слухає порт `80`, перенаправляє запити на порт `5200`).
* **Керування сервісом:** Systemd Unit з підтримкою **Socket Activation**.
* **Користувач сервісу:** Застосунок працює від імені системного користувача `app`.
* **Автоматизація:** Скрипт автоматичного розгортання має налаштовувати середовище "з нуля", включаючи створення користувачів (`student`, `teacher`, `operator`, `app`) та файл `/home/student/gradebook` із вмістом `6`.

### 4. Ролі користувачів (RBAC)
* **student/teacher:** Повний адміністративний доступ (sudo).
* **operator:** Обмежений доступ. Дозволено лише:
    * `systemctl start/stop/restart/status mywebapp.service`
    * `nginx -s reload`
